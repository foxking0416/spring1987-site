教學1：累計成本、累計損益
累計成本是把當支股票所有的交易都累積起來計算的結果
這個數字對長期持有者來說會更有意義
有些人比較無法理解的是，為什麼手上已無該檔股票的庫存了，但依舊有累計成本這個數字呢?
假設目前手上已無該支股票的庫存，那麼累計成本等於直接告知該支股票的累計損益為何
舉個簡單例子，我今天用1000元買一張股票，隔天用1500元賣一張股票，我的累計成本就是-500元
累計成本是負的，就代表一定是獲利狀態。
累計成本也區分為"會扣除股利"跟"不會扣除股利"兩種
一般來說，成本扣除股利更能夠實際顯示出損益狀態
買進：增加成本
賣出：減少成本
增資認購：增加成本
現金減資：減少成本
虧損減資：成本不變
配股配息：減少成本
股票分割/反分割：成本不變



教學2：自動帶入股利、手動輸入股利、混和模式
多數狀況使用"自動帶入股利"即可，不但不需自己上網查詢股利資料，也可避免手動輸入股利資料時，因為每股股利小數點的誤差 例：手機查詢A公司配股0.6元，實際上是0.5999999999
造成最終實領股利的落差。

手動輸入股利的使用情境：
本程式會根據所領股利自動計算補充保費，但因為補充保費只考慮股利裡的『盈餘分配』的部分
有許多的ETF所配發的股利除了該ETF持有公司的盈餘分配以外，還包含了類似『利息收入』或『收益平準金』等項目
而盈餘分配以外的這些項目是不列入補充保費的計算的。
因此目前ETF計算出來的補充保費是有可能有些許落差的。若希望要記錄的非常正確，就需要依靠手動輸入股利的方式來設定補充保費。

混和模式：
某些公司在未公開發行的狀況下也有機會讓員工認股，而在未公發的狀況下配發的股利，程式是撈不到資料的，所以就需要使用者自行輸入，
但也許過了幾年後公司上市了，程式撈的到資料，因此後續年份的股利資料就不需要使用者自行輸入了。
在這種狀況下就很適合使用混和模式。

教學7：補充保費的計算
當股利超過20000元就需要扣繳補充保費
補充保費有兩個費率，2013年到2020年為1.91%，2021年開始調整為2.11%，本程式會根據股利配發日期自動調整稅率計算
補充保費的計算需考慮許多條件，除了20000元股利的門檻以外，還要考慮到股利的來源
例如有些公司今年配發7元現金股利，假設此時手邊有7張股票，原以為的補充保費就是
$7 x 7000 x 2.11% = $1034
但實際上7元現金股利只有5元是來自於盈餘分配，另外2元則是來自於資本公積或法定盈餘公積，而這部分是不用放進補充保費的計算的
也就是說實際上的補充保費會是
$5 x 7000 x 2.11% = $739

除了現金股利以外，股票股利也需要考慮進補充保費裡，以113年的鈊象來說
該年度每股配發10元股票股利，35元現金股利
持有一張股票須扣的補充保費為
(股票股利部分)$10 x 1000 x 2.11% = $211
(現金股利部分)$35 x 1000 x 2.11% = $739
須留意的是，鈊象當時股價破1000元，所以10元的股票股利等於是配一張股票，在除權的時候是要除掉數十萬元現值的
但在計算補充保費時就是以10元下去計算

本程式無法完全自動計算的補充保費是ETF的部分
因為許多ETF在配息的時候除了該ETF持有公司的盈餘分配以外，還包含了類似『利息收入』或『收益平準金』等項目
而盈餘分配以外的這些項目是不列入補充保費的計算的。
因此目前ETF計算出來的補充保費是有可能有些許落差的。若希望要記錄的非常正確，就需要依靠手動輸入股利的方式來設定補充保費。






教學9：增資認購
"增資認購"是給幾種情境使用的：
1. 要增資的公司寄了通知單給你，告知你可以認購的股數，讓你去ATM匯款。匯款完成後，隔一段時間，股票就會撥進你的交割戶裡。
2. 任職的公司可以讓你用較低的價格認購公司股票。
3. 收到他人贈與的股票。
4. 該公司一開始未公開發行，是用私下募資認購的，後來才上市櫃。
總之以上這幾種方式認購的股票都不是透過劵商的系統成交的，所以股票雖然會撥進你的交割戶，你的App也應該可以看到股數的增加，但是App計算的成本會是0元。
本程式的設計在使用增資認購時，#現股成本 會是0元，但是 #累計成本 會是認購的金額。以此來符合各位App顯示的帳目以及實際的損益狀況。
以圖例來說，增資認購花了 $58900，但App會將他的成本當成 0 元，所以未實現損益會高達 $79149，但真正實際上的損益是 $20249 這也是單看劵商App 無法知道的數字
